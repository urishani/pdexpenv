This section lists the different error messages you can get from the system after compiling the code you created. If there is an error - the system stops checking the code and sends an error message.
 
Compliation errors happen when the system "reads" something and does not know how to use it. The error message will tell you where (on what line of the code) it had a problem. It will tell you only about the first problem it finds, other errors will be detected only after correcting the problem and compiling again.
 
In some cases, the problem may be caused by something missing (like an ; at an end of a statement) This might cause the message to point to a line that is ok, if the missing item is added elsewhere.
 
Errors can happen when:  
*
You make a mistake in writing a word in the language.
for example: If you write tbale instead of table , you will have an error.
The error message tells what are the things the system expects to see in [brackets] and prints the part of the line where the error occured.
 
*
If you are using a statement that needs specific words in it, like an itterative statement, omitting or mistyping one of these words, will give you an error. like using looop instead of loop, or forgetting to use loop at all.
